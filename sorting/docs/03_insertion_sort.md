# 03. Insertion Sort (삽입 정렬)

## 문제 정의

주어진 배열을 오름차순으로 정렬하라.

```
정렬된 부분 배열에 새 원소를 올바른 위치에 삽입하는 과정을 반복한다.
카드를 정리하는 것처럼, 각 원소를 적절한 위치에 끼워넣는다.
```

소규모 데이터나 거의 정렬된 데이터에서 매우 효율적이다.

---

## 접근법 1: 기본 삽입 정렬

왼쪽부터 하나씩 정렬된 부분에 삽입합니다.

```python
def insertion_sort_basic(arr: List[int]) -> List[int]:
```

### 라인별 설명

```python
result = arr[:]
n = len(result)
```
- 원본 배열을 보호하기 위해 복사본을 생성합니다.

```python
for i in range(1, n):
    key = result[i]
    j = i - 1
```
- `i=1`부터 시작합니다. 첫 번째 원소는 이미 정렬된 것으로 간주합니다.
- `key`: 현재 삽입할 원소를 임시 저장합니다.
- `j`: 정렬된 부분의 마지막 인덱스입니다.

```python
    while j >= 0 and result[j] > key:
        result[j + 1] = result[j]
        j -= 1
    result[j + 1] = key
```
- `result[j] > key`인 동안: 원소를 한 칸씩 오른쪽으로 밀어냅니다.
- `j < 0`이거나 `result[j] <= key`이면 반복이 종료됩니다.
- `result[j+1] = key`: key를 올바른 위치에 삽입합니다.

### 실행 예제: arr = [12, 11, 13, 5, 6]

```
초기: [12, 11, 13, 5, 6]

i=1, key=11:
  j=0: 12>11 → [12, 12, 13, 5, 6], j=-1
  삽입: [11, 12, 13, 5, 6]

i=2, key=13:
  j=1: 12<13 → 이동 없음
  삽입: [11, 12, 13, 5, 6]

i=3, key=5:
  j=2: 13>5 → [11, 12, 13, 13, 6], j=1
  j=1: 12>5 → [11, 12, 12, 13, 6], j=0
  j=0: 11>5 → [11, 11, 12, 13, 6], j=-1
  삽입: [5, 11, 12, 13, 6]

i=4, key=6:
  j=3: 13>6 → [5, 11, 12, 13, 13], j=2
  j=2: 12>6 → [5, 11, 12, 12, 13], j=1
  j=1: 11>6 → [5, 11, 11, 12, 13], j=0
  j=0: 5<6 → 이동 없음
  삽입: [5, 6, 11, 12, 13]

결과: [5, 6, 11, 12, 13]
```

## 접근법 2: 재귀 삽입 정렬

앞의 n-1개를 먼저 재귀적으로 정렬한 후, n번째 원소를 삽입합니다.

```python
def insertion_sort_recursive(arr: List[int], n: int = None) -> List[int]:
```

### 라인별 설명

```python
if n is None:
    arr = arr[:]
    n = len(arr)
if n <= 1:
    return arr
```
- 최초 호출 시 복사본을 생성합니다.
- **Base Case**: 크기 1 이하이면 이미 정렬됨.

```python
insertion_sort_recursive(arr, n - 1)
key = arr[n - 1]
j = n - 2
while j >= 0 and arr[j] > key:
    arr[j + 1] = arr[j]
    j -= 1
arr[j + 1] = key
```
- 먼저 앞의 `n-1`개 원소를 재귀적으로 정렬합니다.
- 그런 다음 `n`번째 원소를 정렬된 부분에 삽입합니다.
- 삽입 로직은 기본 삽입 정렬과 동일합니다.

### 실행 예제: arr = [3, 1, 2]

```
호출 (n=3): 먼저 (n=2) 재귀 호출
  호출 (n=2): 먼저 (n=1) 재귀 호출
    호출 (n=1): Base case → [3, 1, 2]
  key=1, 삽입 → [1, 3, 2]
key=2, 삽입 → [1, 2, 3]

결과: [1, 2, 3]
```

## 접근법 3: 이진 탐색 삽입 정렬

삽입 위치를 이진 탐색으로 찾아 비교 횟수를 줄입니다.

```python
def insertion_sort_binary_search(arr: List[int]) -> List[int]:
```

### 라인별 설명

```python
for i in range(1, n):
    key = result[i]
    pos = bisect.bisect_left(result, key, 0, i)
```
- `bisect.bisect_left`: 정렬된 부분 `[0, i)` 에서 `key`가 들어갈 위치를 **이진 탐색**으로 찾습니다.
- 비교 횟수가 O(i) → O(log i)로 줄어듭니다.

```python
    for j in range(i, pos, -1):
        result[j] = result[j - 1]
    result[pos] = key
```
- 삽입 위치 `pos`부터 현재 위치 `i`까지 원소를 한 칸씩 뒤로 이동합니다.
- key를 `pos` 위치에 삽입합니다.
- **주의**: 이동(shift) 자체는 여전히 O(i)이므로 전체 시간복잡도는 변하지 않습니다.

### 실행 예제: arr = [5, 3, 4, 1]

```
i=1, key=3:
  bisect_left([5], 3, 0, 1) → pos=0
  이동: [5, 5, 4, 1] → 삽입: [3, 5, 4, 1]

i=2, key=4:
  bisect_left([3, 5], 4, 0, 2) → pos=1
  이동: [3, 5, 5, 1] → 삽입: [3, 4, 5, 1]

i=3, key=1:
  bisect_left([3, 4, 5], 1, 0, 3) → pos=0
  이동: [3, 3, 4, 5] → 삽입: [1, 3, 4, 5]

결과: [1, 3, 4, 5]
```

## 복잡도 비교

| 접근법 | 시간 복잡도 (최선) | 시간 복잡도 (평균) | 시간 복잡도 (최악) | 공간 복잡도 |
|--------|-------------------|-------------------|-------------------|------------|
| 기본 삽입 정렬 | O(n) | O(n²) | O(n²) | O(1) |
| 재귀 삽입 정렬 | O(n) | O(n²) | O(n²) | O(n) 호출 스택 |
| 이진 탐색 삽입 정렬 | O(n log n) 비교 | O(n²) 이동 | O(n²) 이동 | O(1) |
