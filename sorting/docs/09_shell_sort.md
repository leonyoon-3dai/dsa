# 09. Shell Sort (셸 정렬)

## 문제 정의

주어진 배열을 오름차순으로 정렬하라.

```
간격(gap)을 점차 줄여가며 삽입 정렬을 반복 적용한다.
큰 간격으로 먼 원소를 빠르게 이동시킨 후, 간격을 줄여 세밀하게 정렬한다.
```

삽입 정렬의 개선 버전으로, 간격 수열의 선택에 따라 성능이 달라진다.

---

## 접근법 1: 기본 셸 정렬 (Shell의 원래 간격)

간격을 n/2에서 시작하여 절반씩 줄입니다.

```python
def shell_sort_basic(arr: List[int]) -> List[int]:
```

### 라인별 설명

```python
result = arr[:]
n = len(result)
gap = n // 2
```
- 원본 배열의 복사본을 생성합니다.
- 초기 간격을 배열 길이의 절반으로 설정합니다.

```python
while gap > 0:
    for i in range(gap, n):
        temp = result[i]
        j = i
        while j >= gap and result[j - gap] > temp:
            result[j] = result[j - gap]
            j -= gap
        result[j] = temp
    gap //= 2
```
- **외부 루프**: 간격이 0보다 큰 동안 반복합니다.
- **내부 루프**: gap 간격으로 떨어진 원소들에 대해 삽입 정렬을 수행합니다.
- `result[j - gap] > temp`인 동안 원소를 gap만큼 뒤로 밀어냅니다.
- `gap //= 2`: 간격을 절반으로 줄입니다.
- 마지막에 `gap=1`이면 일반 삽입 정렬과 동일하지만, 이전 패스에서 대부분 정렬이 되어 있어 빠릅니다.

### 실행 예제: arr = [23, 12, 1, 8, 34, 54, 2, 3]

```
n=8, 초기 gap=4:

gap=4: [23, 12, 1, 8, 34, 54, 2, 3]
  i=4: temp=34, 23<34 → 이동 없음 → [23, 12, 1, 8, 34, 54, 2, 3]
  i=5: temp=54, 12<54 → 이동 없음 → [23, 12, 1, 8, 34, 54, 2, 3]
  i=6: temp=2, 1<2 → 이동 없음   → [23, 12, 1, 8, 34, 54, 2, 3]
  실제로는 gap=4 간격 서브배열들: [23,34], [12,54], [1,2], [8,3]
  → [23, 12, 1, 3, 34, 54, 2, 8]

gap=2: gap=2 간격으로 삽입 정렬
  → [1, 3, 2, 8, 23, 12, 34, 54]

gap=1: 일반 삽입 정렬 (거의 정렬된 상태)
  → [1, 2, 3, 8, 12, 23, 34, 54]

결과: [1, 2, 3, 8, 12, 23, 34, 54]
```

## 접근법 2: Knuth 간격 수열

h = 3h + 1 수열을 사용합니다 (1, 4, 13, 40, 121, ...).

```python
def shell_sort_knuth(arr: List[int]) -> List[int]:
```

### 라인별 설명

```python
gap = 1
while gap < n // 3:
    gap = gap * 3 + 1
```
- Knuth 간격 수열을 생성합니다: 1, 4, 13, 40, 121, 364, ...
- `n // 3`보다 작은 범위에서 가장 큰 간격을 찾습니다.

```python
while gap >= 1:
    for i in range(gap, n):
        temp = result[i]
        j = i
        while j >= gap and result[j - gap] > temp:
            result[j] = result[j - gap]
            j -= gap
        result[j] = temp
    gap //= 3
```
- 간격을 `gap //= 3`으로 줄입니다 (역으로 h = (h-1) / 3).
- 삽입 정렬 로직은 기본 셸 정렬과 동일합니다.
- **Knuth 수열의 장점**: Shell의 원래 수열(n/2)보다 평균적으로 더 빠릅니다.

### 실행 예제: n=20일 때 간격 수열

```
gap 증가: 1 → 4 → 13 → 40 (40 >= 20//3=6 이므로 13에서 중단)
실제 사용 간격: 13 → 4 → 1

gap=13: 13칸 떨어진 원소끼리 삽입 정렬
gap=4:  4칸 떨어진 원소끼리 삽입 정렬
gap=1:  일반 삽입 정렬 (거의 정렬된 상태)
```

## 접근법 3: Hibbard 간격 수열

2^k - 1 수열을 사용합니다 (1, 3, 7, 15, 31, ...).

```python
def shell_sort_hibbard(arr: List[int]) -> List[int]:
```

### 라인별 설명

```python
gaps = []
k = 1
while (2 ** k - 1) < n:
    gaps.append(2 ** k - 1)
    k += 1
```
- Hibbard 간격 수열을 미리 생성합니다: 1, 3, 7, 15, 31, 63, ...
- `2^k - 1`이 배열 크기보다 작은 동안 간격을 추가합니다.

```python
for gap in reversed(gaps):
    for i in range(gap, n):
        temp = result[i]
        j = i
        while j >= gap and result[j - gap] > temp:
            result[j] = result[j - gap]
            j -= gap
        result[j] = temp
```
- 가장 큰 간격부터 역순으로 처리합니다.
- **Hibbard 수열의 장점**: 최악의 경우 시간복잡도가 O(n^(3/2))로, Shell 수열의 O(n²)보다 개선됩니다.

### 실행 예제: n=16일 때 간격 수열

```
gaps = [1, 3, 7, 15]
(15 < 16이므로 포함, 31 >= 16이므로 중단)

실제 사용 간격 (역순): 15 → 7 → 3 → 1

gap=15: 15칸 떨어진 원소끼리 정렬 (거의 2개씩)
gap=7:  7칸 떨어진 원소끼리 정렬
gap=3:  3칸 떨어진 원소끼리 정렬
gap=1:  일반 삽입 정렬
```

## 복잡도 비교

| 접근법 | 시간 복잡도 (최선) | 시간 복잡도 (평균) | 시간 복잡도 (최악) | 공간 복잡도 |
|--------|-------------------|-------------------|-------------------|------------|
| Shell 간격 (n/2) | O(n log n) | O(n^(3/2)) 추정 | O(n²) | O(1) |
| Knuth 간격 (3h+1) | O(n log n) | O(n^(3/2)) 추정 | O(n^(3/2)) | O(1) |
| Hibbard 간격 (2^k-1) | O(n log n) | O(n^(5/4)) 추정 | O(n^(3/2)) | O(1) |
