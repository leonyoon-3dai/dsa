# 06. Longest Increasing Subsequence (최장 증가 부분 수열)

## 문제 정의

정수 배열 `nums`가 주어질 때, **가장 긴 순증가 부분 수열**의 길이를 구하라.

- 부분 수열: 원래 순서를 유지하며 일부 원소를 선택
- 순증가: 선택한 원소들이 엄격하게 증가
- LeetCode #300

**예시**: [10, 9, 2, 5, 3, 7, 101, 18] → LIS = [2, 3, 7, 101] (길이 4)

---

## 핵심 아이디어

`dp[i]` = `nums[i]`를 **마지막 원소**로 하는 LIS의 길이

각 원소에 대해 이전의 모든 원소를 확인하고, 더 작은 값이 있으면 그 LIS를 이어받습니다.

---

## 코드 라인별 설명

```python
if not nums:
    return 0
```
- 빈 배열이면 0을 반환합니다.

```python
n = len(nums)
```
- 배열 길이를 저장합니다.

```python
dp = [1] * n
```
- 모든 원소는 자기 자신만으로 길이 1의 수열을 만들 수 있으므로 1로 초기화합니다.

```python
for i in range(1, n):
```
- 1번 인덱스부터 순회합니다 (0번은 이미 dp[0]=1).

```python
    for j in range(i):
```
- i 이전의 모든 원소 j를 확인합니다.

```python
        if nums[j] < nums[i]:
            dp[i] = max(dp[i], dp[j] + 1)
```
- `nums[j] < nums[i]`: j번째 원소가 i번째보다 작으면 증가 관계 성립
- `dp[j] + 1`: j에서 끝나는 LIS에 nums[i]를 추가하면 길이가 1 증가
- `max(dp[i], ...)`: 여러 j 중 가장 긴 LIS를 선택

```python
return max(dp)
```
- dp 배열의 최댓값 = 전체 LIS 길이

---

## 실행 예제 1: nums = [10, 9, 2, 5, 3, 7, 101, 18]

```
인덱스:  0   1   2   3   4   5    6    7
값:     10   9   2   5   3   7  101   18
dp:      1   1   1   1   1   1    1    1  (초기)

i=1 (9):  j=0: 10 < 9? No → dp = [1, 1, ...]
i=2 (2):  j=0: 10 < 2? No, j=1: 9 < 2? No → dp[2] = 1
i=3 (5):  j=2: 2 < 5? Yes → dp[3] = max(1, 1+1) = 2
i=4 (3):  j=2: 2 < 3? Yes → dp[4] = max(1, 1+1) = 2
i=5 (7):  j=2: 2 < 7 → dp[5] = 2
           j=3: 5 < 7 → dp[5] = max(2, 2+1) = 3
           j=4: 3 < 7 → dp[5] = max(3, 2+1) = 3
i=6 (101): j=5: 7 < 101 → dp[6] = max(1, 3+1) = 4
i=7 (18):  j=5: 7 < 18 → dp[7] = max(1, 3+1) = 4

최종 dp: [1, 1, 1, 2, 2, 3, 4, 4]
결과: max(dp) = 4
LIS: [2, 5, 7, 101] 또는 [2, 3, 7, 101] 등
```

## 실행 예제 2: nums = [0, 1, 0, 3, 2, 3]

```
i=0: dp[0] = 1
i=1: 0 < 1 → dp[1] = 2
i=2: dp[2] = 1 (0은 아무것보다 크지 않음)
i=3: 0 < 3, 1 < 3, 0 < 3 → dp[3] = max(2, 3, 2) = 3
i=4: 0 < 2, 1 < 2, 0 < 2 → dp[4] = 3
i=5: 0 < 3, 1 < 3, 0 < 3, 2 < 3 → dp[5] = max(2, 3, 2, 4) = 4

결과: 4 → [0, 1, 2, 3]
```

## 실행 예제 3: nums = [5, 4, 3, 2, 1] (내림차순)

```
어떤 원소도 이전 원소보다 크지 않음
dp = [1, 1, 1, 1, 1]
결과: 1 (각 원소 자체가 LIS)
```

## 복잡도

| 항목 | 값 |
|------|-----|
| 시간 복잡도 | O(n^2) |
| 공간 복잡도 | O(n) |

> 참고: 이진 탐색을 사용하면 O(n log n)으로 최적화 가능
