# 07. Edit Distance (편집 거리)

## 문제 정의

두 문자열 `word1`과 `word2`가 주어질 때, word1을 word2로 변환하기 위한 **최소 연산 횟수**를 구하라.

허용 연산:
1. **삽입** (Insert): 문자 하나를 추가
2. **삭제** (Delete): 문자 하나를 제거
3. **교체** (Replace): 문자 하나를 다른 문자로 변경

- LeetCode #72

---

## 핵심 아이디어

`dp[i][j]` = word1의 처음 i글자를 word2의 처음 j글자로 변환하는 최소 연산 횟수

- 문자가 같으면: `dp[i][j] = dp[i-1][j-1]` (추가 연산 불필요)
- 문자가 다르면: 삽입, 삭제, 교체 중 최소 + 1

---

## 코드 라인별 설명

```python
m = len(word1)
n = len(word2)
```
- 두 문자열의 길이를 저장합니다.

```python
dp = [[0] * (n + 1) for _ in range(m + 1)]
```
- `(m+1) x (n+1)` 크기의 2D 배열을 생성합니다.

```python
for i in range(m + 1):
    dp[i][0] = i
```
- **첫 번째 열 초기화**: word1[:i]를 빈 문자열로 변환 = i번 삭제
- 예: "abc" → "" = 3번 삭제

```python
for j in range(n + 1):
    dp[0][j] = j
```
- **첫 번째 행 초기화**: 빈 문자열을 word2[:j]로 변환 = j번 삽입
- 예: "" → "abc" = 3번 삽입

```python
for i in range(1, m + 1):
    for j in range(1, n + 1):
```
- 모든 (i, j) 쌍에 대해 순회합니다.

```python
        if word1[i - 1] == word2[j - 1]:
            dp[i][j] = dp[i - 1][j - 1]
```
- 두 문자가 **같으면**: 추가 연산이 필요 없습니다.
- 이전 상태(dp[i-1][j-1])를 그대로 가져옵니다.

```python
        else:
            dp[i][j] = 1 + min(
                dp[i][j - 1],      # 삽입
                dp[i - 1][j],      # 삭제
                dp[i - 1][j - 1]   # 교체
            )
```
- 문자가 **다르면** 세 가지 연산 중 최소를 선택하고 +1합니다:
  - `dp[i][j-1] + 1`: word2에 문자 **삽입** (j를 1 줄임)
  - `dp[i-1][j] + 1`: word1에서 문자 **삭제** (i를 1 줄임)
  - `dp[i-1][j-1] + 1`: 문자를 **교체** (둘 다 1 줄임)

## 실행 예제 1: "horse" → "ros"

```
       ""  r  o  s
""      0  1  2  3
h       1  1  2  3
o       2  2  1  2
r       3  2  2  2
s       4  3  3  2
e       5  4  3  3

결과: 3회
```

**연산 과정:**
1. horse → rorse (h를 r로 **교체**)
2. rorse → rose (r **삭제**)
3. rose → ros (e **삭제**)

## 복잡도

| 항목 | 값 |
|------|-----|
| 시간 복잡도 | O(m x n) |
| 공간 복잡도 | O(m x n), 최적화 시 O(min(m, n)) |
